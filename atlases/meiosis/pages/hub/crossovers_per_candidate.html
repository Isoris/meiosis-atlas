▰ Crossovers — per inversion candidate (ideogram · telomere bias · optional PRDM9 logo)
Stage: hub · spec / phase C · driven by crossover_track · optional: prdm9_motif · keyed by shared.candidate
What this page shows
spec · phase C
Three views over the recombination map around one inversion candidate at a time, pared down from the aye-aye CO paper's Fig.1 (Versoza et al. GBE 2024). The active candidate is read cross-atlas from shared.candidate; the renderer fetches the candidate-keyed JSON (data/crossovers/<candidate_id>.json) and renders the three panels scoped to that candidate's span + its flanks.
The PRDM9 motif logo is facultative: when the prdm9_motif field is null/missing the renderer hides the logo card. NCO + gene-conversion tracts live on the sibling NCO per-candidate page rather than overlaid here.
Migrated 2026-05-19 from genome-atlas page11. Per-candidate meiosis views belong in the Meiosis Atlas alongside the cohort-level crossover scaffold; cross-atlas references stay (the Genome Atlas's chromosome-overview CO sub-track now points here).

View 1 — sex-specific CO ideogram (candidate ± flank)
primary view
One vertical rod per chromosome in the candidate's scope: the host chrom plus any chromosomes referenced by the candidate's flanking windows. CO events as dots beside the rod: females on the left, males on the right. The candidate's span is drawn as a translucent band on the host rod.

Translucent band = the candidate's inverted span. Inside the band recombination is expected to be suppressed; dots inside the band are the read-out of how strong that suppression actually is.
View 2 — CO rate vs. relative telomere distance (candidate scope)
primary view
x-axis: relative distance from telomere (0 = telomere, 1 = centre of chromosome arm). y-axis: # COs per individual per bin. One smoothed curve per sex (red / blue) with a 95% CI band — computed from the candidate-scoped events, optionally pre-binned in telomere_curve.

View 3 — PRDM9 binding motif optional
hidden when no pwm
When prdm9_motif.pwm is present (an N×4 ACGT position-weight matrix), the renderer draws a sequence logo (bits on y-axis, position on x-axis). When the field is null/missing the renderer sets display:none on this card.

Import schema — crossover_track.json (per inversion candidate)
contract
One JSON per inversion candidate. candidate_id must match an entry in the Inversion Atlas's candidates_registry. events are the CO calls scoped to the candidate's span + flank (flank_bp, default 5 Mb). All optional fields may be omitted entirely — the corresponding panel self-hides.

One file per inversion candidate, named by candidate_id: data/crossovers/<candidate_id>.json. The renderer never enumerates the directory — it only fetches the file matching the active candidate.
