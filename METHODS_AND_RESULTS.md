# Methods & Results: A statistical pipeline for inferring inversion effects on meiotic recombination from pedigree-derived crossover and gene-conversion calls

## Provenance note

This document describes the mathematics and statistics of an analysis chain
whose **server-side compute is implemented in-repo** (pure Python modules under
`atlases/meiosis/registries/runners/`) and whose **window-level recombination
observables are implemented in-repo in JavaScript** (`atlases/meiosis/shared/`).

Two provenance boundaries matter for the reader:

1. **Upstream / external.** The per–departure-interval event classes (NCO, CO,
   DCO, MOSAIC_SHORT, MOSAIC_LONG, AMBIG, LOW_CONFIDENCE) are produced by an
   external classifier (ngsTracts, run over ngsPedigree Stage-3 departure
   intervals). This repository only ingests and type-coerces that output; the
   classification decision rule itself is **not** in this repository and must be
   confirmed against the ngsTracts source. Likewise the per-(chromosome×dyad)
   event-count table and the inversion candidate intervals are produced
   upstream; this repository consumes them.
2. **In-repo, but not yet run on the cohort.** Every estimator below is
   implemented and unit-tested against synthetic fixtures, but **no estimator
   has been executed on the 226-sample cohort** — there is no result artifact on
   disk (the data directory `atlases/meiosis/data/` contains only a `.gitkeep`;
   there is no workspace `layers/` or `raw_results/` tree). The window-level
   JavaScript observables read a synthetic `DEMO` constant
   (`atlases/meiosis/shared/recomb_data.js:5`), not cohort data. Consequently
   the Results section reports only the one class of artifact that genuinely
   exists on disk — the machine-readable analysis registry — and explicitly
   enumerates what was **not** produced.

All formulas below were read directly from source and are cited `path:line`.

## Abstract

**Cohort / data.** A 226-sample hatchery pedigree of *Clarias gariepinus*
(reference `fClaHyb_Gar_LG`), reduced upstream to per–departure-interval
meiotic-event classifications and to per-(chromosome×dyad) crossover counts
(`README.md:148`).

**Problem.** Chromosomal inversions are hypothesized to suppress crossing-over
both locally (inside a heterokaryotypic inversion, on the inversion's own
chromosome) and, more controversially, on *other* chromosomes ("interchromosomal
effect"). Testing this from pedigree data is hard for three reasons: the
precondition (double-strand break) and resolution (crossover vs non-crossover)
layers are not separately identifiable from offspring genotypes; crossover
counts per dyad are small and over-dispersed; and inversion carriers cluster in
families, so a naive carrier-vs-noncarrier contrast measures family structure
rather than the inversion.

**Method chain.** Raw event classes are type-coerced into typed per-interval and
per-(chromosome×dyad) tables; from these the pipeline computes (i) a cohort-level
2×2 enrichment of short gene-conversion tracts inside vs outside inversions by
Fisher's exact test; (ii) a per-candidate version of the same enrichment with
Benjamini–Hochberg control across candidates; (iii) a per-chromosome
heterozygote-vs-homozygote crossover-rate contrast by Welch's *t*; and (iv) the
headline interchromosomal test, a Welch *t* statistic on per-parent crossover
rates whose null distribution is generated by a **family-aware permutation**
(karyotype labels shuffled only within family blocks) and corrected across
off-focal chromosomes by Benjamini–Hochberg and Bonferroni. A separate
window-level layer expresses the Sandler two-step interference model as
eligibility (NCO/Mb), resolution (CO share), and the coefficient of coincidence
*C* = r_ij / (r_i·r_j).

**Headline result.** None on the cohort: no estimator has been run on the 226
samples, and no result artifact exists on disk. The only on-disk analytical
artifact is the registry that declares the chain — 13 modules / 12 analyses / 12
analysis-modes / 12 output layers — all four statistical chains marked
implemented-and-dispatchable but not yet executed.

## 1. Introduction

The biological target is the effect of a chromosomal inversion on meiotic
recombination. Inside a heterokaryotypic inversion, single crossovers produce
inviable recombinant gametes, so crossing-over is expected to be suppressed
while non-crossover gene conversion can persist; a short gene-conversion tract
(here the class `MOSAIC_SHORT`) inside an inversion is therefore the
characteristic local signature. The more demanding claim is that a focal
inversion alters recombination on chromosomes other than its own.

Three statistical obstacles shape every estimator in this pipeline. First,
**non-identifiability**: from offspring genotypes one observes resolved
crossovers and gene-conversion tracts, but the latent precondition probability
and exchange probability of the Sandler two-step model are not separable; the
code documents this explicitly and works only with observable proxies
(`atlases/meiosis/shared/recomb_data.js:4`). Second, **small, sparse counts**:
per-dyad crossover counts are integers near zero, so the comparison primitives
are exact (Fisher) or small-sample-robust (Welch's *t* with
Welch–Satterthwaite degrees of freedom) rather than asymptotic. Third,
**family confounding**: inversion carriers concentrate in family hubs, so the
null model must preserve family structure; this pipeline does so by permuting
karyotype labels only within family blocks
(`atlases/meiosis/registries/runners/meiosis_interchromosomal_effect.py:167`,
`atlases/meiosis/shared/inversion_meiosis.js:238`).

The remainder is organized along the data flow. Section 2 covers input encoding
and type coercion. Section 3 gives the reusable comparison primitives (Fisher's
exact test; Welch's *t* with a hand-rolled *t*-distribution tail). Section 4 is
the cohort-level gene-conversion enrichment. Section 5 is its per-candidate
fan-out with multiplicity control. Section 6 is the per-chromosome
intrachromosomal crossover-rate contrast. Section 7 is the headline
interchromosomal test with its family-aware permutation null and
multiple-comparison correction. Section 8 covers the derived rate fields
(per-Mb crossover rate; coefficient of coincidence) and the window-level
interference observables. Section 9 covers the focal-inversion coincidence
scan and its causal-ladder gating. Section 10 covers serialization and
reproducibility.

## 2. Input encoding, type coercion, and windowing

The pipeline does not call variants or classify events; it ingests an external
classifier's output and renders it into typed records. Two ingest contracts
exist.

**2.1 Per–departure-interval tract records.** Each interval carries a class
∈ {NCO, CO, DCO, MOSAIC_SHORT, MOSAIC_LONG, AMBIG, LOW_CONFIDENCE}
(`atlases/meiosis/registries/extractors/normalize_tract_classifications.py:49`),
genomic coordinates (`chrom`, `start`, `end`, `span_bp`), and an
`inside_inversion` flag ∈ {yes, partial, no}. Coercion maps a fixed set of null
sentinels — `{"", "NA", "NaN", "-", "null", "None"}` — to missing
(`normalize_tract_classifications.py:47`), so the upstream "-" placeholder is not
mistaken for data. Integers and floats are parsed; unrecognized class strings
are dropped. This stage is **adapter-only**: the classification itself is
external.

**2.2 Per-(chromosome×dyad) event counts.** Each row carries `parent_id`,
`offspring_id`, `chrom`, `chrom_len_bp`, integer counts `n_co`, `n_dco`,
`n_nco`, an optional per-Mb rate, and an optional karyotype label at a focal
inversion ∈ {homA, het, homB}
(`atlases/meiosis/registries/extractors/normalize_chromosome_meiosis_events.py:30`).
When a per-Mb rate is absent but a count and chromosome length are present, the
rate is **derived** (Section 8.1).

**2.3 Windowing.** The window-level observables operate on fixed-width windows
of one chromosome, each carrying a population-layer NCO count (`n_NCO_pop`), a
pedigree-layer crossover count (`n_CO_ped`), and a constant window width in Mb
(`atlases/meiosis/shared/recomb_data.js:8`). In the current code these windows
come from the synthetic `DEMO` object; the header states the helpers are
designed to be repointed at a real NCO-tract envelope without changing the math
(`atlases/meiosis/shared/recomb_data.js:13`).

## 3. Reusable comparison primitives

### 3.1 Fisher's exact test on a 2×2 table

For a table [[a,b],[c,d]] the probability of any table with the same margins is
the hypergeometric weight, computed in log space for stability via `lgamma`:

```
log P(a,b,c,d) = lgamma(a+b+1) + lgamma(c+d+1) + lgamma(a+c+1) + lgamma(b+d+1)
               - lgamma(n+1) - lgamma(a+1) - lgamma(b+1) - lgamma(c+1) - lgamma(d+1)
```

(`atlases/meiosis/registries/runners/meiosis_nco_enrichment.py:27`). The
enumeration runs a′ over `max(0, r1+c1−n) … min(r1, c1)` with the other three
cells determined by the fixed margins
(`meiosis_nco_enrichment.py:79`). Two p-values are returned:

- **Two-sided** — the sum (log-sum-exp) of probabilities of all tables at most
  as probable as the observed table, with a tie tolerance of 1e-9
  (`meiosis_nco_enrichment.py:92`).
- **One-sided (greater)** — the tail sum over a′ ≥ a, used as the directional
  enrichment p-value (`meiosis_nco_enrichment.py:97`).

The odds ratio is (a·d)/(b·c), returned as +∞ when b·c = 0 and a·d > 0, and as
NaN for a fully degenerate table (any zero margin)
(`meiosis_nco_enrichment.py:70`). A Haldane-corrected log-odds,
`log(((a+0.5)(d+0.5))/((b+0.5)(c+0.5)))`, is reported as a finite diagnostic
(`meiosis_nco_enrichment.py:75`).

### 3.2 Welch's two-sample *t* and a hand-rolled *t* tail

For two samples the statistic uses unbiased per-group variances
(`meiosis_intrachromosomal_co.py:121`) and unequal-variance standard error:

```
t  = (mean_x − mean_y) / sqrt(var_x/n_x + var_y/n_y)
df = (var_x/n_x + var_y/n_y)^2
     / [ (var_x/n_x)^2/(n_x−1) + (var_y/n_y)^2/(n_y−1) ]   (Welch–Satterthwaite)
```

(`meiosis_intrachromosomal_co.py:150`, df at `:154`). The test returns NaN when
either group has n < 2 or both variances vanish
(`meiosis_intrachromosomal_co.py:133`).

Because the runtime has no scientific-stats dependency, the Student-*t* CDF is
built from the regularized incomplete beta function I_x(a,b), evaluated by
Lentz's continued fraction (`meiosis_intrachromosomal_co.py:31`, convergence
eps = 1e-15, max 200 iterations) with the standard symmetry flip at
x > (a+1)/(a+b+2) (`meiosis_intrachromosomal_co.py:86`). The CDF uses the identity

```
F(t; df) = 1 − ½·I_x(df/2, ½),   x = df/(df + t²)   for t ≥ 0
```

(`meiosis_intrachromosomal_co.py:97`), and the two-sided p is
2·(1 − F(|t|; df)) (`meiosis_intrachromosomal_co.py:102`). This is the standard
recipe the external `scipy.special.betainc` implements; the implementation is
in-repo and was verified in testing against reference *t* CDF values and the
internal-consistency identity `p_two_sided == 2·(1 − CDF(|t|))`.

### 3.3 Multiplicity control

Benjamini–Hochberg is a step-up over the finite p-values: sort ascending, set
q_(r) = min(running, p_(r)·m/r) scanning from the largest rank down so the
adjusted values are monotone, where m is the number of finite p-values
(`meiosis_interchromosomal_effect.py:226`, core step at `:238`). Bonferroni is
min(1, p·m) with the same finite-count m (`meiosis_interchromosomal_effect.py:247`).
Non-finite inputs are preserved as NaN at their original positions in both.

## 4. Cohort-level gene-conversion enrichment inside vs outside inversions

This stage asks whether a target gene-conversion class (default `MOSAIC_SHORT`;
`meiosis_nco_enrichment.py:20`) is enriched inside inversions relative to the
rest of the NCO-like population. The population is restricted to NCO-like tracts,
i.e. class ∈ {NCO, MOSAIC_SHORT} (`meiosis_nco_enrichment.py:24`); CO, DCO,
AMBIG and LOW_CONFIDENCE are excluded, as are tracts whose `inside_inversion`
is neither "yes" nor "no" (the "partial" calls are conservatively dropped)
(`meiosis_nco_enrichment.py:138`). The 2×2 table is

|                       | inside inversion | outside inversion |
|-----------------------|------------------|-------------------|
| target class          | a                | b                 |
| other NCO-like        | c                | d                 |

(`meiosis_nco_enrichment.py:141`). Fisher's exact test (Section 3.1) gives the
two-sided and one-sided-greater p-values and the odds ratio; the one-sided p is
the manuscript-relevant enrichment statistic. The result is a single cohort-level
record (`meiosis_nco_enrichment.py:160`).

## 5. Per-candidate enrichment with multiplicity control

The cohort test is fanned out to one test per inversion candidate. For each
candidate the tract population is restricted to the candidate's **own
chromosome**, and "inside" means the tract interval overlaps the candidate span
by at least 1 bp:

```
intersects = (t_chrom == c_chrom) AND NOT (t_end < c_start OR t_start > c_end)
```

(`atlases/meiosis/registries/runners/meiosis_nco_per_candidate.py:30`). The same
NCO-like restriction applies (`meiosis_nco_per_candidate.py:73`); tracts with
missing coordinates are excluded and counted
(`meiosis_nco_per_candidate.py:78`). Each candidate yields a 2×2 table with the
same cell semantics as Section 4 but with "outside" meaning outside-the-span
on the same chromosome (`meiosis_nco_per_candidate.py:82`), and Fisher's exact
test is applied per candidate (`meiosis_nco_per_candidate.py:159`). Candidates
lacking `chrom`/`start_bp`/`end_bp` are emitted as `skipped` with null
statistics so the output row count equals the candidate-set size
(`meiosis_nco_per_candidate.py:58`).

Across candidates, the per-candidate one-sided p-values are corrected by
Benjamini–Hochberg and Bonferroni (Section 3.3), and a candidate is flagged
significant when its BH-adjusted p is below α (default 0.05;
`meiosis_nco_per_candidate.py:103`, flag at `:178`). This stage performs a
**cross-source join**: tract records from this repository's domain against
inversion candidate intervals produced upstream; the candidate intervals are
external and the join is purely coordinate-based.

## 6. Intrachromosomal crossover-rate contrast (heterozygote vs homozygote)

For each chromosome the pipeline contrasts the per-dyad crossover rate between
dyads heterozygous at the focal inversion and dyads homozygous (the homA and
homB labels are merged into a single non-het group;
`meiosis_intrachromosomal_co.py:24`, merge at `:196`). The per-row rate is the
supplied `co_per_mb` or, if absent, derived as n_co / chrom_len_bp · 1e6
(`meiosis_intrachromosomal_co.py:168`). Rows lacking a karyotype label or a
derivable rate are skipped (`meiosis_intrachromosomal_co.py:186`).

Per chromosome, Welch's *t* (Section 3.2) compares the het rate sample to the
non-het rate sample; chromosomes with fewer than two dyads in either group are
emitted with null statistics and `excluded_reason = "insufficient_dyads"`
(`meiosis_intrachromosomal_co.py:219`). The reported effect size is the rate
ratio mean_het / mean_non_het, and a chromosome is flagged when this ratio falls
below a suppression threshold (default 0.7; `meiosis_intrachromosomal_co.py:23`,
flag at `:238`). Note this stage applies no across-chromosome multiplicity
correction — the per-chromosome two-sided Welch p-values are reported as-is
(`meiosis_intrachromosomal_co.py:250`).

## 7. Headline: the interchromosomal effect test

This is the manuscript headline — does a focal inversion's karyotype alter
crossover rates on chromosomes *other* than its own?

**7.1 Per-parent rate.** For each (parent, chromosome) the counts are summed and
divided by chromosome length: rate = (Σ n_co [+ Σ n_dco]) / chrom_len_bp · 1e6,
with single-crossover-only the default (include_co = true, include_dco = false;
`meiosis_interchromosomal_effect.py:112`, division at `:138`).

**7.2 Observed statistic.** For a given focal inversion, parents are labeled by
karyotype (het vs non-het = homA∪homB; `meiosis_interchromosomal_effect.py:146`)
and, for each tested chromosome, Welch's *t* (Section 3.2) is computed on the
per-parent rates (`meiosis_interchromosomal_effect.py:339`). The focal
inversion id defaults to the first id (sorted) present in the design table when
unspecified (`meiosis_interchromosomal_effect.py:299`).

**7.3 Family-aware permutation null.** The null distribution of the *t*
statistic is generated by shuffling karyotype labels **within permutation
blocks only** (the block is typically the family;
`meiosis_interchromosomal_effect.py:167`), via a Fisher–Yates shuffle restricted
to each block (`meiosis_interchromosomal_effect.py:184`). This preserves the
per-family karyotype composition, so the null measures the inversion rather than
the family. The permutation p-value uses add-one smoothing:

```
p = (1 + #{ |t_perm| ≥ |t_obs| }) / (N_finite + 1)
```

(`meiosis_interchromosomal_effect.py:217`), with N = 10000 permutations by
default (`meiosis_interchromosomal_effect.py:283`). The shuffle is driven by a
mulberry32 PRNG that reproduces the JavaScript implementation bit-for-bit when
seeded, so a seeded run is exactly reproducible
(`meiosis_interchromosomal_effect.py:40`).

**7.4 Multiplicity and the focal-chromosome exclusion.** The focal inversion's
own chromosome is reported but **excluded from the alpha control**, because the
biological question concerns the other chromosomes
(`meiosis_interchromosomal_effect.py:356`). Benjamini–Hochberg and Bonferroni
(Section 3.3) are applied across the off-focal-chromosome permutation p-values,
and a chromosome is flagged significant when its BH-adjusted p is below α
(default 0.05; `meiosis_interchromosomal_effect.py:285`). A per-chromosome
"local-inversion burden" (count and total length of other inversions on the
tested chromosome) is attached as a covariate flag but is **not** entered into a
regression in this version (`meiosis_interchromosomal_effect.py:95`).

## 8. Derived rate fields and the window-level interference observables

### 8.1 Per-Mb rates

When a producer omits the per-Mb rate, both the events normalizer and the chain
runners derive it identically as count / chrom_len_bp · 1e6
(`normalize_chromosome_meiosis_events.py:105`). Missing inputs or a non-positive
length yield null rather than zero, to distinguish "no data" from "zero rate".

### 8.2 Coefficient of coincidence

Two coincidence computations exist, with the same definition C = r_ij/(r_i·r_j):

- **Interval-pair (adapter) form.** C = r_ab / (r_a·r_b), derived when the
  producer omits it and r_a·r_b > 0, else null
  (`atlases/meiosis/registries/extractors/normalize_coincidence_matrix.py:120`).
  Pairs with C above a negative-interference threshold (default 3.0;
  `normalize_coincidence_matrix.py:56`) are flagged as artefact-likely.
- **Window-pair form.** Single-window rate r = n_CO_ped / n_meioses
  (`atlases/meiosis/shared/recomb_data.js:45`), double rate r_ij = n_DCO /
  n_meioses, and C = r_ij/(r_i·r_j), with C undefined when either single rate is
  zero (`recomb_data.js:50`). Interference is I = 1 − C
  (`recomb_data.js:59`).

### 8.3 Eligibility and resolution maps

Two further per-window observables express the Sandler two-step layers:

- **Eligibility** (precondition / DSB-substrate proxy): NCO events per Mb,
  n_NCO_pop / window_mb (`recomb_data.js:33`).
- **Resolution** (exchange proxy): the crossover share given a precondition,
  n_CO_ped / (n_NCO_pop + n_CO_ped), returned as undefined when the window total
  is below a minimum sample size (default 3; `recomb_data.js:39`).

These window-level observables are method-complete but read synthetic `DEMO`
windows in the current code (Section 1, provenance boundary 2).

## 9. Focal-inversion coincidence scan and causal-ladder gating

A second, carrier-based formulation of the interchromosomal question is
implemented in `atlases/meiosis/shared/inversion_meiosis.js`. For a focal
inversion, individuals are split into carriers and non-carriers, and for each
tested chromosome the difference in coefficient of coincidence,
delta_C = C_carrier − C_control, is computed
(`inversion_meiosis.js:222`, `:225`; delta at `:347`). The null is again
family-aware: carrier/non-carrier labels are shuffled **within family hubs**,
preserving each hub's carrier count (`inversion_meiosis.js:241`), and the
two-sided permutation p uses the same add-one smoothing, (n_extreme+1)/(n_perm+1),
with n_perm = 1000 by default (`inversion_meiosis.js:267`, p at `:278`).

The per-row verdict is a guarded classifier (`inversion_meiosis.js:371`):

```
hub carrier-share ≥ 0.80                       → FAMILY CONFOUNDED
p < 0.01 and |delta_C| ≥ 0.30                  → STRONG EFFECT
p < 0.05 and |delta_C| ≥ 0.15                  → MODERATE EFFECT
p < 0.05                                        → WEAK EFFECT
otherwise                                       → no/!effect
```

The "family confounded" guard fires first, encoding the warning that a result
driven by a single family hub is uninterpretable as an inversion effect. A
"causal ladder" gate further requires a raw |delta_C| ≥ 0.15 association
(level 0→1 gate; `inversion_meiosis.js:515`) and then a family-aware
p_perm < 0.05 (`inversion_meiosis.js:520`) before a row is allowed to ascend.
This formulation reads synthetic `DEMO` karyotype, triad, and family structures
(`inversion_meiosis.js:43`, `:64`, `:111`) and is therefore METHOD-only on the
cohort.

## 10. Serialization and reproducibility

Every chain emits a typed result payload (`per_chrom`/`per_candidate`/`result`
plus a `summary` block) with non-finite values coerced to null so the JSON is
strict-mode serializable
(`meiosis_interchromosomal_effect.py:323`,
`meiosis_intrachromosomal_co.py:211`). Each result carries provenance: the
source layer ids, the parameters used, and the module name and version. The
randomized chain (Section 7) is exactly reproducible under a fixed integer seed
because the permutation PRNG is a deterministic mulberry32
(`meiosis_interchromosomal_effect.py:40`); without a seed it falls back to the
platform RNG (`meiosis_interchromosomal_effect.py:290`).

The set of analyses, the module backing each, the declared output layer of each,
and the input dimensions each requires are serialized as a machine-readable
registry (five JSONL tables) by
`atlases/meiosis/registries/generate_catalogue_outbound.py`, which enforces three
referential-integrity constraints at generation time: every analysis-mode's
analysis type resolves to a declared analysis; every mode's `produces` is
single-valued and is among that analysis's declared outputs; and every mode's
module name resolves to a declared module
(`atlases/meiosis/registries/generate_catalogue_outbound.py` — constraints
re-checked by `atlases/meiosis/registries/test_catalogue_outbound.py`).

### 11.2 What was NOT produced on this data

No estimator in Sections 3–9 was executed on the 226-sample cohort, and there is
no corresponding result artifact:

- The data directory contains only a placeholder
  (`atlases/meiosis/data/.gitkeep`); there is no per-interval tract envelope, no
  per-(chromosome×dyad) event envelope, no coincidence matrix, no inversion
  candidate envelope, and no workspace `layers/` or `raw_results/` tree on disk.
- Therefore there is **no** cohort value for the cohort gene-conversion
  enrichment odds ratio or p (§4), **no** per-candidate p-value table (§5),
  **no** per-chromosome het-vs-non-het rate ratio or Welch p (§6), and **no**
  interchromosomal permutation p-values or BH-significant chromosome set (§7).
- The window-level eligibility / resolution / coincidence / interference maps
  (§8.2–8.3) and the carrier-based focal scan (§9) read a synthetic `DEMO`
  object, not cohort data, so they likewise have no cohort result.

Accordingly, no numeric finding is reported as a result. The estimators have
been exercised only on synthetic unit-test fixtures (deterministic by seed),
which verify correctness of the mathematics but are not cohort data and are not
reported here as results.
